## Purpose

Define how security issues are reported and what response boundaries apply.

## Policy

- Provide a clear reporting path for vulnerabilities.
- State which versions or environments are supported.
- Keep the wording public-safe and operationally actionable.
- Avoid exposing incident-sensitive implementation details.

- The repository-instance security surface lives in the root `SECURITY.md`.
- This policy is the reusable quality-layer counterpart for downstream repositories.
