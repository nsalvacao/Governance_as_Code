# Artifact / Build Provenance Record

One record per built artifact. Complete all sections at build time and store the record alongside or linked from the artifact. The SLSA framework requires that provenance be generated by a trusted build system and not by the artifact's author.

## Artifact Identification

Uniquely identifies the artifact. The digest is the canonical immutable reference.

- Artifact ID: `{{ARTIFACT_ID}}` (unique identifier assigned by the build system)
- Artifact name: `{{ARTIFACT_NAME}}` (human-readable name, e.g., `my-service-linux-amd64`)
- Version: `{{VERSION}}` (semantic version or build number, e.g., `1.4.2` or `build-4821`)
- Digest: `{{DIGEST}}` (content-addressable hash, e.g., `sha256:abc123…`; this is the primary immutable reference)

## Build Provenance

Describes the build system that produced the artifact. SLSA Level 2+ requires the builder to be a hosted, authenticated service.

- Build ID: `{{BUILD_ID}}` (unique ID for this build run, e.g., GitHub Actions run ID)
- Builder identity: `{{BUILDER_ID}}` (e.g., `https://github.com/actions/runner` for GitHub Actions)
- Build system: `{{BUILD_SYSTEM}}` (e.g., GitHub Actions, Jenkins, Tekton, Google Cloud Build)
- Builder trust level: `{{BUILDER_TRUST}}` (internal classification: untrusted / hosted-shared / hosted-isolated)

## Source Provenance

Links the artifact back to the exact source code that produced it. All fields are mandatory for SLSA Level 1+.

- Source repository: `{{SOURCE_REPO}}` (full URL of the repository)
- Source commit: `{{SOURCE_COMMIT}}` (full SHA-1 or SHA-256 commit hash; no abbreviated hashes)
- Branch: `{{BRANCH}}` (branch at build time, e.g., `main`, `release/1.4`)
- Tag: `{{TAG}}` (release tag if applicable, e.g., `v1.4.2`; leave blank for non-release builds)

## SLSA Level

Declare the SLSA level this build achieves. Level requirements are cumulative.

- Achieved SLSA level: `{{SLSA_LEVEL}}` (1 / 2 / 3 / 4)
- Level-specific requirements met:
  - Level 1: Provenance exists and is generated by the build system.
  - Level 2: Hosted build service; provenance is authenticated and non-falsifiable by the developer.
  - Level 3: Hardened build platform; no external influence on the build; two-party review on source.
  - Level 4: Hermetic, reproducible build; full dependency verification.

## Build Environment

Characterises the reproducibility and isolation properties of the build environment.

- Hermetic (no network access during build): `{{HERMETIC}}` (yes / no)
- Reproducible (same inputs produce identical output): `{{REPRODUCIBLE}}` (yes / no)
- Dependencies (list all direct build-time dependencies with verified hashes):

| Dependency | Version | Hash |
|---|---|---|
| `{{DEPENDENCY_1}}` | `{{DEPENDENCY_VERSION_1}}` | `{{DEPENDENCY_HASH_1}}` |
| `{{DEPENDENCY_N}}` | `{{DEPENDENCY_VERSION_N}}` | `{{DEPENDENCY_HASH_N}}` |

## Attestation

Provenance must be signed and verifiable. SLSA Level 2+ requires the attestation to be generated by the build system, not the developer.

- Attestation format: `{{ATTESTATION_FORMAT}}` (e.g., SLSA provenance v0.2, in-toto statement, Sigstore bundle)
- Signing key ID: `{{SIGNING_KEY_ID}}` (reference to the key used to sign; do not record the key value here)
- Attestation storage location: `{{ATTESTATION_STORAGE}}` (e.g., OCI registry with cosign, Rekor transparency log)

## Verification Instructions

Document the exact command a downstream consumer must run to verify this artifact's provenance before use.

```
{{VERIFY_COMMAND}}
```

Example: `cosign verify-attestation --type slsaprovenance --certificate-identity {{BUILDER_ID}} {{ARTIFACT_NAME}}@{{DIGEST}}`
