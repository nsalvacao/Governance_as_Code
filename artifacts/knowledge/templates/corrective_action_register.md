## Purpose

Track action items that arise from incidents, reviews, and analysis so that improvement work stays visible and owned.

## Required fields

- Action ID
- Description
- Source event
- Owner
- Priority
- Due date
- Status
- Verification method

## Guidance

- Use one row per action.
- Make the verification method explicit.
- Prefer measurable completion criteria over vague closure language.
