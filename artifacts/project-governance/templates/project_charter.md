# Project Charter / Project Brief

## Purpose

Formally authorize the project, define its scope, and identify the accountable owner.

## Fields

- project name
- sponsor
- project manager or owner
- purpose and scope
- constraints and exclusions
- high-level milestones
- approval line

## Usage

1. Use this as the entry point for project governance.
2. Keep scope boundaries explicit to reduce downstream ambiguity.
3. Cross-link the charter to the business case and initiation plan.
