# Issue Log / Issue Register

## Purpose

Track issues that need action, decision, or escalation.

## Fields

- issue id
- date raised
- description
- impact
- priority
- owner
- status
- resolution date

## Usage

1. Log each issue once and update its status through the same record.
2. Differentiate open issues from risks and change requests.
3. Link to evidence and follow-up actions when available.
