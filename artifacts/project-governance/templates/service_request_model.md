# Service Request Model / Request Catalog

## Purpose

Define the request types that can be fulfilled in a repeatable way.

## Fields

- request type
- request description
- eligibility
- fulfillment steps
- owner
- approval requirements
- expected turnaround

## Usage

1. Use one entry per repeatable request type.
2. Keep fulfillment steps standardized enough to automate.
3. Link request types to the service catalog and known error records where relevant.
