# Service Catalog / Service Portfolio Record

## Purpose

Describe the services offered, their ownership, scope, and status.

## Fields

- service name
- service description
- owner
- consumer group
- status
- dependencies
- service level reference

## Usage

1. Keep the catalog as the authoritative inventory of services.
2. Distinguish live services from planned or retired entries.
3. Link each service to its request model, SLA, and configuration record.
